# Clinical Signal: Dynamic Intake Architecture## Executive SummaryClinical Signal requires a flexible intake system that accepts any patient data type—video calls, audio notes, PDFs, Word docs, images, and structured forms. This document outlines a production-grade architecture that:- Routes file types to appropriate processors (transcription, PDF extraction, OCR)- Maintains HIPAA compliance by minimizing external API calls- Scales from 2 alpha users (10 uploads/month) to 1,000 production users (5,000/month)- Costs ~$50/month at alpha, ~$350/month at beta, ~$8,000/month at production- Keeps PHI in-network by defaulting to open-source + self-hosted models---## 1. For alpha, this costs ~$0 beyond server fees. If practitioners request higher accuracy (especially medical terminology), implement AssemblyAI as a fallback with a "high-accuracy mode" toggle.**Self-Hosting Strategy:**- Use [faster-whisper](https://github.com/SYSTRAN/faster-whisper) (optimized CPU inference via CTranslate2)- For GPU: Deploy on Railway ($10–20/month) or RunPod ($0.34/hour on RTX 4090)- Small/turbo models (74M/140M params) run on CPU in ~2x real-time; medium needs GPU- Result: $10–50/month for MVP, zero per-minute transcription costs---### Document Parsing & OCR| Service | Use Case | Cost | HIPAA | Notes ||---------|----------|------|-------|-------|| **PyMuPDF** | Native PDFs with simple text | Free | Self-hosted | Fast, but fails on scanned/complex docs || **Amazon Textract** | Forms, tables, handwriting, scanned PDFs | $0.01/page | Yes (SOC2, HIPAA-eligible) | Handles medical forms well; $1–3/month for alpha || **Tesseract** (open-source) | OCR fallback | Free | Self-hosted | Good for clean documents; struggles with noise || **Google Cloud Document AI** | Premium accuracy, complex layouts | $1.50/1K pages | Enterprise BAA possible | Better accuracy than Textract; pricier |**Recommendation:** Hybrid approach:1. Try PyMuPDF first (free, fast)2. If PyMuPDF fails (returns <30% of expected text), fall back to Textract3. For OCR on scanned images, use Tesseract (local) or Textract**Cost at scale:**- Alpha (10 PDFs/month): ~$0.10- Beta (50 PDFs/month): ~$0.50- Production (5,000 PDFs/month): ~$50---### Embeddings & Vector Search| Model | Cost | Latency | Accuracy | Notes ||-------|------|---------|----------|-------|| **text-embedding-3-small (OpenAI)** | $0.02/1M tokens | 1–3s | High | Use batch API for 50% discount || **Cohere embed-4** | $0.01/1M tokens | <1s | High | Cheapest; 768-dim vectors || **self-hosted BAAI/bge-small-en** | $0 (compute only) | 2–5s | High | On Railway: $10/month + GPU |**Recommendation:** Use OpenAI's batch API with `text-embedding-3-small` for MVP (discounted to $0.01/1M tokens). At 10,000 pages × 300 tokens avg = 3M tokens/month, cost is ~$0.03/month. Self-hosting becomes cost-effective only above 50–100M tokens/month.---## 2. Processing Pipeline Architecture```┌─────────────────────────────────────────────────────────────────┐│ PRACTITIONER UPLOADS FILE                                       ││ (Vercel: POST /api/intake/upload)                              │└────────────────┬────────────────────────────────────────────────┘                 │                 ▼┌─────────────────────────────────────────────────────────────────┐│ 1. DETECT FILE TYPE & VALIDATE                                  ││    - MIME type check (magic bytes)                              ││    - File size (max 500MB for video, 50MB for docs)            ││    - Virus scan (ClamAV local or Cloudmersive API)             ││    - Return 202 Accepted with job_id                            │└────────────────┬────────────────────────────────────────────────┘                 │                 ▼┌─────────────────────────────────────────────────────────────────┐│ 2. QUEUE ASYNC JOB (Inngest)                                    ││    - Save metadata: file_id, patient_id, file_type, size       ││    - Queue document to processing pipeline                      ││    - Set status = "processing"                                  │└────────────────┬────────────────────────────────────────────────┘                 │        ┌────────┴────────┬──────────────┬──────────────┐        │                 │              │              │        ▼                 ▼              ▼              ▼    ┌────────┐      ┌──────────┐  ┌──────────┐  ┌──────────┐    │ VIDEO  │      │ AUDIO    │  │ PDF/DOCX │  │ IMAGE    │    │        │      │          │  │          │  │          │    └────┬───┘      └────┬─────┘  └────┬─────┘  └────┬─────┘         │               │             │             │         ▼               ▼             ▼             ▼    ┌─────────────────────────────────────────────────────┐    │ 3. PROCESS BY FILE TYPE (Railway background worker) │    │                                                      │    │ VIDEO: ffmpeg extract audio → Whisper              │    │ AUDIO: Whisper (or AssemblyAI fallback)            │    │ PDF:   PyMuPDF → Textract fallback                 │    │ DOCX:  mammoth → plain text                        │    │ IMAGE: Tesseract OCR (or Textract)                 │    │ FORM:  Parse structured JSON                       │    └──────────┬──────────────────────────────────────────┘               │               ▼    ┌─────────────────────────────────────────────────────┐    │ 4. EXTRACT TEXT + METADATA                          │    │    - Raw text (stored in text_content column)       │    │    - Speaker diarization (if audio/video)          │    │    - Timestamps or page references                  │    │    - Confidence scores (if available)              │    └──────────┬──────────────────────────────────────────┘               │               ▼    ┌─────────────────────────────────────────────────────┐    │ 5. CHUNK & EMBED TEXT (Inngest Job)                │    │    - Split on sentence boundaries (~300 tokens)     │    │    - Batch embed via OpenAI batch API (cheaper)    │    │    - Store vectors in pgvector                      │    │    - Index: HNSW (M=16, ef_construction=200)       │    └──────────┬──────────────────────────────────────────┘               │               ▼    ┌─────────────────────────────────────────────────────┐    │ 6. PRESENT TO PRACTITIONER                          │    │    - Dashboard shows extracted text                 │    │    - Flag low confidence sections for review        │    │    - Allow inline edits + save corrections          │    │    - Mark as "verified" when ready                  │    └──────────┬──────────────────────────────────────────┘               │               ▼    ┌─────────────────────────────────────────────────────┐    │ 7. STORE FOR PROTOCOL GENERATION                    │    │    - intake_documents table (raw + extracted)       │    │    - Create embeddings for semantic search          │    │    - Available for AI analysis pipeline             │    └─────────────────────────────────────────────────────┘```### Key Design Decisions**Why Inngest for job queue?** - Vercel serverless functions timeout at 60 sec (Pro) / 5 sec (Hobby)- Inngest provides durable, retry-enabled background jobs without worker infrastructure- Native Vercel integration; no Redis/Postgres queue to manage- Free tier: 1,000 events/month (adequate for alpha)**Why Rails/Render for processing?** - Vercel can't run CPU-heavy tasks (Whisper, Textract polling, embedding)- Railway/Render are cheaper than AWS + support long-running processes- Alternative: AWS Batch (auto-scales, pay-per-use) for production**Storage Strategy:**- Small files (<5MB): Upload to S3, store S3 path + metadata in PostgreSQL- Large files (>5MB): Stream directly to S3 via presigned URL (avoid 4.5MB body limit)- Extracted text: Stored in `intake_documents.extracted_text` column (full-text search)- Embeddings: Stored in pgvector (`intake_documents.embedding` VECTOR(1536))**HIPAA Compliance:**- Never send full files to third-party APIs; send only after text extraction locally (where possible)- Textract is SOC 2 / HIPAA-eligible; request BAA- Rev.ai and AssemblyAI offer BAAs; include BAA requirement in contract- Local Whisper avoids BAA entirely (no third-party processing)---## 3. Database Schema```sql-- New tables for dynamic intakeCREATE TABLE intake_documents (  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),  patient_id UUID NOT NULL REFERENCES patients(id) ON DELETE CASCADE,  file_type VARCHAR(50), -- 'video', 'audio', 'pdf', 'docx', 'image', 'form'  original_filename VARCHAR(500),  s3_key VARCHAR(1000), -- s3://bucket/path/file  file_size_bytes INT,    -- Processing status  processing_status VARCHAR(50) DEFAULT 'pending', -- pending, extracting, extracted, embedding, complete, failed  error_message TEXT,    -- Extracted data  extracted_text TEXT, -- Full transcribed or OCR'd text  metadata JSONB, -- {pages: 5, duration_seconds: 3600, speakers: [...], confidence: 0.95}  extracted_at TIMESTAMP,    -- Embeddings  embedding VECTOR(1536), -- pgvector for semantic search  embedded_at TIMESTAMP,    -- Source tracking  uploaded_at TIMESTAMP DEFAULT NOW(),  created_by UUID NOT NULL REFERENCES users(id),  updated_at TIMESTAMP DEFAULT NOW(),    -- Practitioner review  reviewed_by UUID REFERENCES users(id),  reviewed_at TIMESTAMP,  is_verified BOOLEAN DEFAULT FALSE,  corrections_made TEXT);CREATE INDEX idx_intake_patient ON intake_documents(patient_id);CREATE INDEX idx_intake_status ON intake_documents(processing_status);CREATE INDEX idx_intake_embedding ON intake_documents USING HNSW (embedding vector_cosine_ops) WITH (m=16, ef_construction=200);-- Track chunks for RAGCREATE TABLE document_chunks (  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),  document_id UUID NOT NULL REFERENCES intake_documents(id) ON DELETE CASCADE,  chunk_index INT,  text_content TEXT NOT NULL,  embedding VECTOR(1536),  token_count INT,    -- For video/audio: time range  start_seconds FLOAT,  end_seconds FLOAT,    -- For PDFs: page number  page_number INT,    created_at TIMESTAMP DEFAULT NOW());CREATE INDEX idx_chunks_document ON document_chunks(document_id);CREATE INDEX idx_chunks_embedding ON document_chunks USING HNSW (embedding vector_cosine_ops);-- Processing jobs table (for audit + debugging)CREATE TABLE processing_jobs (  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),  document_id UUID NOT NULL REFERENCES intake_documents(id),  job_type VARCHAR(50), -- 'transcribe', 'extract_pdf', 'ocr', 'embed'  inngest_event_id VARCHAR(500),  status VARCHAR(50),  result JSONB,  duration_ms INT,  started_at TIMESTAMP,  completed_at TIMESTAMP);```---## 4. Build Order & Implementation Plan### Phase 1: Foundation (Weeks 1–2)1. **Database schema** — Add `intake_documents`, `document_chunks`, `processing_jobs` tables2. **File upload API** — Vercel serverless: `POST /api/intake/upload` (file validation, S3 storage, Inngest queueing)3. **Local Whisper deployment** — Set up Railway instance with `faster-whisper`; test with 1-minute audio sample**Cost:** ~$20/month (Railway small instance + testing)### Phase 2: Processing Pipeline (Weeks 3–4)4. **Inngest integration** — Background jobs for transcription, PDF extraction, embedding5. **PyMuPDF + Textract hybrid** — Implement graceful fallback for complex PDFs6. **Tesseract OCR** — Docker container on Railway for image text extraction7. **Batch embeddings** — OpenAI batch API integration (cheaper than real-time)**Cost:** ~$35/month (add Textract, batch job failures, testing)### Phase 3: UI & Review (Weeks 5–6)8. **Extraction review dashboard** — Display extracted text, flag low-confidence sections, allow corrections9. **Correction workflow** — Practitioners edit extracted text; corrections update pgvector embeddings10. **Document linking** — Associate intake_documents with patient profiles; display in patient timeline**Cost:** ~$15/month (minimal additional compute)### Phase 4: Advanced Features (Weeks 7+)11. **Speaker diarization** — For call recordings (Whisper limitation; use Rev.ai or AssemblyAI fallback)12. **Semantic search** — Use pgvector to find related documents across patient history13. **Automated tagging** — Use Claude to tag extracted documents (symptoms, labs, timeline events)**Cost:** ~$50–100/month (depends on feature; defer to future sprints)---## 6. HIPAA Compliance Notes### Data Residency- **Local Whisper:** Audio never leaves your infrastructure ✓- **PyMuPDF + Tesseract:** Document processing is local ✓- **OpenAI embeddings:** Tokenization happens at API; request data residency if available- **Textract:** Requires SOC 2 audit; AWS data residency is regional ✓- **Rev.ai / AssemblyAI:** Require signed BAAs; audio is processed on their servers### Recommended SetupFor MVP with maximum privacy:- Use local Whisper for all audio/video- Use PyMuPDF for simple PDFs, Tesseract for OCR- Only use Textract for complex forms (request BAA)- Avoid Rev.ai/AssemblyAI unless practitioners specifically request higher accuracy**Cost savings:** ~$0/month for transcription (vs. $900+/month if sending all audio to APIs)### BAA Requirements (if using third-party APIs)- Rev.ai: Offers BAA at enterprise level; ~$500–2,000/month minimum- AssemblyAI: Offers BAA; included with paid plans- Textract: AWS BAA available via AWS Business Associate Addendum- OpenAI: No standard BAA for general API; must negotiate custom agreement**Recommendation:** Sign BAAs for Rev.ai + Textract if practitioners request medical-grade accuracy. Otherwise, stay local.---## 7. Security Checklist- [ ] All intake_documents have `created_by` (audit trail)- [ ] RLS policy: practitioners can only see their own patients' documents- [ ] Raw files encrypted at rest (S3 default)- [ ] Extracted text encrypted at rest (PostgreSQL with ENC)- [ ] Virus scan all uploads before processing (ClamAV or Cloudmersive)- [ ] No PHI in application logs (use structured logging, redact in handlers)- [ ] Inngest events are encrypted in transit- [ ] Railway / Render instances run behind VPC (not public)- [ ] Textract + Rev.ai calls are logged and audited- [ ] Expiration policy: Delete raw files after 90 days (keep extracted text + embeddings)---## Summary & Recommendations**For MVP (Weeks 1–4):**- Deploy local Whisper on Railway (~$15/month)- Use PyMuPDF + Textract hybrid for PDFs- Use Tesseract for images- OpenAI batch API for embeddings (~$0.01/month at alpha scale)- **Total: ~$35/month, zero per-minute transcription costs****For Beta (Weeks 5+):**- Add speaker diarization via AssemblyAI fallback (optional)- Implement correction workflow- Monitor Rails/Whisper performance; scale if needed- **Total: ~$115/month****For Production:**- Consider Rev.ai if medical accuracy is critical ($900/month for heavy users)- Auto-scale Whisper on Railway to handle spikes- Monitor pgvector query performance; add HNSW indexing- **Total: ~$1,200–5,000/month (depending on transcription volume)****Key Decision: Local Whisper vs. External APIs**- **Local Whisper:** $0/month, best privacy, 94–96% accuracy, no medical terminology- **AssemblyAI:** $0.90/hour, BAA available, 97%+ medical accuracy- **Rev.ai:** $15/hour human, $0.25/min AI, enterprise HIPAA tierRecommend starting with local Whisper, add AssemblyAI as a practitioner-toggled "high-accuracy mode" for complex medical terminology, and defer Rev.ai to enterprise tier.
